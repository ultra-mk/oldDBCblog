Hashes vs. Objects

Wherin I phone it in yet again

May 3, 2015

Warning; editorializing follows. Even before I started Dev Bootcamp, I was somewhat aware that JavaScript had a rough reputation. This curriculum this week has reinforced that idea. There were a few snide remarks in the challenges (e.g. "Do you miss Ruby yet?") and the Google+ community page has it's share of people complaining.

I don't think that's productive. At all. And I kinda wish people would stop it. Is there weirdness is JavaScript? Sure. There's weirdness everywhere. I came from a modest Python background, so the multitude of methods in Ruby is absolutely bonkers to me. I find it hard to read. I spend a lot of time searching for the perfect method because there probably is an exactly perfect method. Python makes way more sense to me. Does that mean Ruby sucks? No. It's just different.

And the other fact of the matter is JavaScript is on practically every computer currently running on the planet. At this moment, I have six tabs in google chrome open. All of the sites on those tabs use JavaScript. I have no idea which ones use Ruby or Python. Knowing JavaScript makes you useful to every company that publishes a website. Bottom line.

My last gripe about this - it's counterproductive. It's like training a mechanic to hate the catalytic converter because of how it's engineered. You're a mechanic. Sometimes you have to fix catalytic converters. Tell me, does it make you a better mechanic to stand there, hands on hips, and say something like "Catalytic converters suck"? No it doesn't. So my modest plea, quit complaining about JavaScript. Saying something sucks does not make you a better developer. If something sucks, we should be getting in there and fucking fixing it. Sorry for the language, but that's how I feel. Stay positive or GTFO.

So rant aside. The assignment this week is to compare an aspect of Ruby with an aspect of JavaScript. Since I have a natural fondness for data structures, I am going to take a look at Ruby hashes compared with JavaScript objects. This first difference is one of syntax.

JavaScript is quite simple. Apologies for the superheroes again, I went to C2E2 last weekend. 


hulk = {
     skin:           "green",
     strength:     "strongest one there is",
     pants:      "purple"
}

And the three ways I could do this in Ruby. First the hashrocket. 


hulk = {
      'skin' => 'green',
     'strength' => 'strongest one there is",
     'pants' => 'purple'
}

Using symbols


hulk = {
     skin: 'green',
     strength: 'strongest one there is',
     pants: 'purple'
}

On the surface, it doesn’t look that different. There are a few key differences. In Ruby you have the option to use symbols as keys. The symbol doesn’t really exist in JavaScript. The keys in a JavaScript object are necessarily strings. The second major difference is how Ruby implements classes. JavaScript is not as strongly class based as Ruby. This means that Ruby hashes tie into their parent class. To me, it seems like JavaScript objects are more self-contained.

Good gravy. Another week of phoning in the blog posts.
