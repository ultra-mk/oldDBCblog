Map Ref. 41°N 93°W

A way of getting from here to there.

April 11, 2015

The map and the each method are closely related. On first glance they look almost identical. 

my_array = [1,2,3]

my_array.each {|num| print num + 1 }
    234=> [1,2,3]

my_array.map {|num| num + 1 }
    #=> [2,3,4]

The crucial difference is that map returns a new object. Each simply executes the code block, displays the results and leaves the original array intact. The .each method technically returns the original array back to the caller. The best explanation I stumbled across is each is used when you want to do something to an array - map is when you want to do something to the array and use the results. I like to think of it as such - a map will take you from one place to another, but regardless of where you are, both places exist.

An example: 

super_heroes = ['Hulk', 'Iron Man', 'Black Widow']

super_heroes.map {|hero| hero + ' is awesome!'}
     #=> ['Hulk is awesome!','Iron man is awesome!','Black Widow is awesome!']

My array is simply a list of super heroes. Then I run the map method and map those values to three sentences. This is where map makes the most sense - in this example it treats the array of super heroes and the array of sentences as two different objects. Grammatically speaking, one array is a list of nouns and the other is a list of sentences. Map also provides a destructive version to permanently modify the original array.
