Git Hub Cheat Sheet

What is Git?

Open source version control

Tracks changes that are made to a file

Allows multiple people to work on a single file

Key commands:

> git init == creates a repository

> git status == tells you what has been added or tracked

> git diff == tesll about the changes to the repository

> git log == history of changes

What is a Branch?

Branching allows multiple possible pathways that you may choose

Acts as a multiverse

> git checkout <BRANCH> == creates a branch or switches to it

> git checkout -b <BRANCH> == creates a branch and switches to it

Forking and Cloning

On the repo to be copied, click on the fork button and choose

On your forked repository copy the clone URL

in the command line type:

> git clone URL

Adding

stages files to be tracked by GIT

> git add --all

or

git add

Commiting

Acts as a checkpoint to return to at any point

> git commit -m "message" == commit the changes

> git commit -v == commit changes with commit message in the file

Tracking Changes

add the the files to track the changes

> git add

to remove trackers:

> git reset HEAD <file>

Making Changes

Any changes you make will be tracked once added

> git status -v == shows the specific changes to be made

Pushing Changes

updates the master branch with commited changes

> git push origin master

Adding remotes

Sets up remote to pull from

> git remote add upstream <FORKED REPO URL>

> git remote -v == to see your remotes

Fetching changes

gets changes made to the master branch

> git fetch upstream <BRANCH>

> git fetch upstream == fetches all the branches

Merging:

takes changes made by 2+ people and combines them

> git merge upstream <BRANCH>

Git pull

combines both fetching changes and merging them

> git pull upstream origin

git pull
